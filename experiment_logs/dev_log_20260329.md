## 本次修改摘要

### 1. ik_server.py — NatureIKSolver.solve_arm 维度修复

**文件**：`ik_server.py`

**问题原因**：
- 训练配置 `n_obs_steps=3`，意味着模型的 `global_cond_dim = n_obs_steps × obs_dim = 3 × 20 = 60`。
- 但 `solve_arm` 在推理时只构造了当前单帧观测，送入 `predict_action` 的 obs 形状为 `(1, 1, 20)`。
- `DiffusionResNetLowdimPolicy.predict_action` 内部执行 `nobs[:, :n_obs_steps]` 取前 3 帧，输入只有 1 帧时 reshape 后得到 `(1, 20)`，而模型期望 `(1, 60)`，导致维度不匹配报错。

**修复方式**：
```python
# 修复前
obs_ts = torch.from_numpy(obs).float().to(self.device).view(1, 1, -1)

# 修复后
n_obs = self.policy.n_obs_steps
obs_ts = torch.from_numpy(obs).float().to(self.device)
obs_ts = obs_ts.unsqueeze(0).unsqueeze(0).repeat(1, n_obs, 1)  # (1, n_obs, D)
```
将单帧重复 `n_obs_steps` 次，使形状对齐为 `(1, 3, 20)`。
